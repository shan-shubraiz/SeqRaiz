üöÄ Running BLAST from the Command Line

BLAST can also be executed directly through the command line (CMD or Terminal), which is faster and more flexible for batch processing. Below are the steps to install and run BLAST locally on Windows.

Step 1: Download BLAST+

  • Go to the official NCBI BLAST+ download page: Download BLAST+
  • Download the ZIP file for your operating system (e.g., win64 for 64-bit Windows).
  • Extract the folder (e.g., ncbi-blast-2.x.x+) to a location such as C:\Program Files\BLAST+.

Step 2: Add BLAST+ to PATH

  • Open System Properties ‚Üí Environment Variables.
  • Find Path, click Edit, and add the bin folder inside your BLAST installation (e.g., C:\Program Files\BLAST+\bin).
  • Open a new CMD window and check installation by typing: 
    blastn -version

Step 3: Prepare Query and Database

  • Place your query file in FASTA format, e.g., query.fasta, inside your working directory.
  • If using a local database, format it first using makeblastdb: 
    makeblastdb -in my_sequences.fasta -dbtype nucl -out mydb
  • If using NCBI‚Äôs remote database, you can use -db nt -remote directly.

Following is the explanation of a command to run BLAST using command prompt terminal> It also explains some alternatives options to get required results of BLAST.

Standard Command 

blastn -query query.fasta -db nr -out results.txt -outfmt 6 -evalue 1e-5 -num_threads 4 -max_target_seqs 10 -word_size 11

Explanation:
This is a standard local BLASTn command comparing nucleotide sequences from query.fasta against a standard database nr, saving results in tabular format. It uses 4 CPU threads, shows the top 10 hits, and uses a default word size of 11 for nucleotide BLAST.

üìù Explanation

  • 1. blastn
    Chooses the BLAST program type.
    Variants:
    ‚Ä¢ blastn ‚Üí Nucleotide vs Nucleotide
    ‚Ä¢ blastp ‚Üí Protein vs Protein
    ‚Ä¢ blastx ‚Üí Translate nucleotide query ‚Üí search protein db
    ‚Ä¢ tblastn ‚Üí Protein query ‚Üí search translated nucleotide db
    ‚Ä¢ tblastx ‚Üí Translated nucleotide vs translated nucleotide

  • 2. -query query.fasta
    Input file containing your query sequence(s) in FASTA format.
    Alternative: Provide a full path or use -query - to paste sequences directly.

  • 3. -db mydb
    Database against which you‚Äôre searching. Must be formatted with makeblastdb if local.
    Alternatives:
    ‚Ä¢ -db nt or -db nr with -remote ‚Üí Use NCBI‚Äôs databases.
    ‚Ä¢ -subject subject.fasta ‚Üí Compare directly to a FASTA file.

  • 4. -out results.txt
    File to save results.
    Alternatives:
    ‚Ä¢ Omit ‚Üí results print directly in CMD.
    ‚Ä¢ Redirect manually: > results.txt.

  • 5. -outfmt 6
    Output format.
    Common choices:
    ‚Ä¢ 0 ‚Üí Pairwise (default, human-readable)
    ‚Ä¢ 5 ‚Üí XML
    ‚Ä¢ 6 ‚Üí Tabular
    ‚Ä¢ 7 ‚Üí Tabular + comments
    ‚Ä¢ 10 ‚Üí CSV
    ‚Ä¢ 11 ‚Üí BLAST archive

  • 6. -evalue 1e-5
    Expect value threshold (smaller = stricter).
    Alternatives:
    ‚Ä¢ -evalue 1e-10 ‚Üí very strict.
    ‚Ä¢ -evalue 10 ‚Üí default, less strict.

  • 7. -num_threads 4
    Number of CPU threads to use.
    Default = 1. Use more if your system supports it.

  • 8. -max_target_seqs 10
    Limit the number of hits shown per query.
    Alternatives:
    ‚Ä¢ 1 ‚Üí Only best hit.
    ‚Ä¢ 500 ‚Üí Extensive results.
    Default = 500.

  • 9. -word_size 11
    Seed word length for finding matches.
    Smaller = more sensitive but slower.
    Common values:
    ‚Ä¢ Nucleotide BLAST ‚Üí default = 11, can reduce to 7 for short queries.
    ‚Ä¢ Protein BLAST ‚Üí default = 3.
